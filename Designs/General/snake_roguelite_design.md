## 1. 游戏概述

### 1.1 一句话描述

一款以贪吃蛇为操作核心的 Roguelite 游戏：玩家操控一条蛇在 Grid 地图上移动，通过收集「蛇鳞」（遗物）构建 Build，在分层地图中完成战斗与探索，最终击败每层 Boss 通关。

### 1.2 核心体验支柱

| 支柱 | 描述 |
|------|------|
| **操作核心** | 贪吃蛇的基础操作规则不变：不能停止、越吃越长、碰自身和边界即死 |
| **Build 构筑** | 通过选择蛇头、蛇尾、蛇鳞的组合，形成不同的游玩风格 |
| **战场交互** | 状态效果存在于蛇身和地板格子上，移动本身就是在改造战场 |
| **风险权衡** | 长度既是生命值又是攻击消耗，所有决策都是关于「值不值得用长度换」 |

### 1.3 关键机制摘要

- **长度 = 唯一的数值单位**：生命值、攻击消耗、战斗资源统一用「格」表示
- **蛇头/蛇尾 = 规则层**：改变游戏规则的运作方式（如「长度缩短」被重新定义为什么）
- **蛇鳞 = 内容层**：描述「当 X 发生时产生 Y 效果」的遗物，装在固定槽位中
- **状态效果 = 空间现象**：附着在蛇身或地板格子上，两者可互相转化，并产生反应
- **分层地图**：每层由多个房间构成，房间之间可反复探索，层内有时间压力

### 1.4 解耦说明

本文档中所有系统均以**数据驱动**的方式描述。以下内容待后续专项设计文档补充，不影响当前系统实现：

- **美术**：所有视觉表现（颜色、粒子、动画）以「需求描述」形式出现，不与逻辑耦合
- **叙事**：所有事件、建筑物的文本内容为占位描述，叙事填充不影响机制逻辑
- **音效**：未在本文档描述，由音效设计师根据触发事件清单独立设计

## 2. 系统一：基础移动系统

### 2.1 系统描述

游戏的操作底层。蛇在离散的 Grid 地图上以固定节奏移动，玩家只能控制方向，无法停止。

### 2.2 设计理念

> 贪吃蛇的操作规则是本游戏所有其他系统的物理基础。所有设计必须在「蛇无法停止」「移动路径即历史」这两个约束下成立。

### 2.3 设计框架

```
Grid 地图
  └─ 每个单元格（Cell）有坐标 (x, y) 和状态（空/蛇身/敌人/地形/状态格）
  
蛇的数据结构
  └─ 有序队列：[蛇头坐标, 身体段1坐标, ..., 蛇尾坐标]
  └─ 每 tick 向当前方向移动一格：头部插入新坐标，尾部弹出旧坐标
  └─ 吃到食物时：头部插入新坐标，尾部不弹出（长度+1）

移动节奏
  └─ 固定 tick 间隔（基础值，可被 buff 修改）
  └─ 转向指令在下一个 tick 生效（非立即生效）
  └─ 同一 tick 内只接受一次转向指令
```

### 2.4 详细设计

**碰撞判定（死亡条件）**

| 碰撞类型 | 默认结果 | 可被修改 |
|----------|----------|----------|
| 蛇头碰边界 | 死亡 | 部分蛇头/鳞片可修改 |
| 蛇头碰自身身体 | 死亡 | 特定鳞片可使某段身体虚化 |
| 蛇头碰敌人 | 直接吞噬敌人（L1：无消耗） | 蛇头 Atom Chain 可改写吃的收益 |
| 敌人攻击蛇身 | 受击计数器+1（累计 3 次丢 1 段） | 蛇头/蛇尾 Atom Chain 可改写阈值 |

**转向规则**

- 不允许 180° 直接反向（标准贪吃蛇规则）
- 转向延迟：默认 0（L1 无转向延迟机制）

**视觉需求（待美术）**

- 蛇头与蛇身需要视觉区分（L1：HEAD=0.95, BODY=0.78, TAIL=0.6 灰度）
- 蛇身每格显示所携带状态的颜色（火=红橙、冰=蓝、毒=绿，无状态=白/灰）
- 蛇尾需要视觉区分（特定蛇尾有特殊外观需求）

## 3. 系统二：长度系统

### 3.1 系统描述

长度是游戏中**唯一的数值单位**，同时扮演生命值、战斗资源、地图空间压力三种角色。

### 3.2 设计理念

> 不引入独立的 HP 系统。「我还有多少格」是玩家唯一需要关注的数字。所有伤害、恢复、增益、减益最终都折算为长度的增减。

### 3.3 设计框架

```
长度 = 蛇身格子总数（包含蛇头和蛇尾）

长度变化来源：
  增加：吃到食物、特定鳞片触发、特定事件奖励
  减少：战斗消耗、状态效果伤害（灼烧等）、特定事件代价

长度阈值（影响游戏体验，不影响规则）：
  危险区    1~4格
  生存区    5~10格
  战斗区    11~20格
  舒适区    21~35格
  过载区    36格+（地图空间压力开始超过战斗压力）

最大长度上限：无硬上限，但过载区的空间压力是软上限
最小长度：1格（蛇头本身），长度归 0 时死亡
```

### 3.4 详细设计

**长度增加规则**

| 来源 | 增加量 | 备注 |
|------|--------|------|
| 食物 | +1 格 | 敌人击杀后散落，蛇头经过时吃掉 |
| 蛇头 Atom（如 Hydra） | +N 格（直接增长） | 跳过食物流程，由 `direct_grow` 原子执行 |
| 猎食间完成 | +2 格（永久） | L2+ 房间系统 |
| 蛇尾 Atom（如 Salamander） | +1 格（窗口内恢复） | 由 `direct_grow` 原子执行 |
| 特定蛇鳞 | 按鳞片 Atom Chain 描述 | 见系统五 |

**长度减少规则**

| 来源 | 减少量 | 备注 |
|------|--------|------|
| 敌人攻击蛇身（累计 3 次命中） | -1 格 | hits_per_segment_loss=3，可被蛇头修改 |
| 状态效果（灼烧） | -1 格/2 秒 | 见系统三（L1 中灼烧仅作用于敌人，不直接扣蛇长度） |
| 特定事件代价 | 按事件描述 | 见系统八 |

> **L1 修订：** 蛇头吃敌人不消耗长度（直接吞噬），敌人死亡后掉落食物。长度减少的唯一来源是敌人攻击蛇身累积受击计数器满后丢段。详见 12A 节。

## 4. 系统三：状态效果系统

> **前置系统：** 系统一（基础移动）、系统二（长度）

### 4.1 系统描述

状态效果是**空间现象**，存在于两种载体上：实体（蛇身段、敌人）和空间（地板格子）。两种载体上的状态可以互相转化，同类或不同类状态相遇时触发反应。

### 4.2 设计理念

> 状态效果不是绑在角色身上的数值标签，而是流动的地形。蛇的移动本身就是在「书写」战场状态。设计新状态效果时，必须同时设计其实体效果、空间效果和转化规则。

### 4.3 设计框架

```
状态效果数据结构：
  type        : 状态类型（枚举：火/冰/毒/酸/电/虚）
  layer       : 层数（整数，1~N，影响效果强度）
  carrier     : 载体类型（实体 / 空间）
  duration    : 持续时间（空间载体用；实体载体按 tick 计算）

转化规则（实体 → 空间）：
  蛇身带状态经过某格 → 在该格生成对应空间状态格
  触发条件由各状态类型单独定义（见 4.4）

转化规则（空间 → 实体）：
  实体移动到状态格 → 实体获得对应状态（层数 +1）

反应触发条件：
  同一实体身上存在两种不同状态 → 触发反应
  地板上两种不同状态格相邻 → 触发反应
  实体带状态进入不同类型的状态格 → 触发反应

反应结果：
  消耗参与反应的两种状态
  产生第三种效果（按反应类型定义）
  反应伤害 = (状态A层数 + 状态B层数) × 基础系数
```

### 4.4 基础状态详细设计

> **L1 已实现：** 火🔥、冰🧊、毒☠️ 三种状态。酸🧪、电⚡、虚👻 为 L2+ 预留设计。

#### 🔥 灼烧 [火] ✅ L1 已实现

| 实体效果 | 每 2 秒 -1 格长度（蛇）/ 部位受 1 次伤害（敌人）；持续 6 秒；可叠层 |
| 空间效果 | 火焰格持续 8 秒；经过的实体获得灼烧（层数+1）；相邻格每秒 20% 概率蔓延 |
| 实体→空间 | 灼烧状态的蛇经过某格时，该格生成火焰格（转移 1 层） |
| 空间→实体 | 实体踩入火焰格，获得灼烧 |
| 蛇的特殊互动 | ~~碾压已移除（L1）~~；灼烧不扩散到整条蛇。火光环对邻格敌人造成火属性伤害，异类状态触发反应 |

#### 🧊 冰冻 [冰] ✅ L1 已实现

| 蛇的特殊互动 | 蛇头冰冻时转向延迟但蛇尾保持原速，蛇身产生堆叠增加自撞风险 |

## 5. 系统四：蛇头／蛇尾系统

> **前置系统：** 系统一、系统二、系统三
> **L2 修订版：** 基于 L1 实际战斗循环（吃敌人无消耗 + 敌人攻击蛇身 + per-segment status）重新设计

### 5.1 系统描述

蛇头和蛇尾是「规则层遗物」。它们不描述效果，而是**改写游戏规则的运作方式**。蛇头定义**攻击风格**（如何吃敌人、吃了之后发生什么），蛇尾定义**防御风格**（被攻击后如何结算、丢段后如何恢复）。

### 5.2 设计理念

> 蛇头/尾决定这一局的「打法风格」。两个不同的蛇头应该让玩家做出完全不同的操作决策——不是"都在吃敌人但一个更强"，而是"吃敌人的节奏和风险模型完全不同"。
>
> **给数值策划的指导：** 蛇头/尾的数值不是效果强度，而是规则的边界参数（如触发阈值、无敌持续时间、受击容忍度）。平衡调整应优先调整这些参数，而非给蛇头/尾加减伤害数值。

### 5.3 设计框架

```
架构：统一 Atom Chain 模式
  蛇头/蛇尾/蛇鳞 三套系统统一使用 T25 Effect Atom System
  每个蛇头/蛇尾 = game_config.json 中一组 EffectChain 定义
  复用 AtomRegistry、AtomExecutor、TriggerManager 基础设施
  新增蛇头/蛇尾 = JSON 配置 + 可能需要的新原子（.gd 文件），不改 Snake 核心代码

蛇头 Atom Chain：绑定 on_kill / on_head_hit 触发器
  改写「吃敌人」的收益和代价
  原子拆解举例：modify_food_drop(0) + direct_grow(1) + steal_status + modify_hit_threshold(-1) = Hydra

蛇尾 Atom Chain：绑定 on_length_decrease / on_interval 触发器
  改写「受伤后」的恢复和结算规则
  原子拆解举例：mark_recovery_window(5s) + if_in_recovery_window → direct_grow(1) = Salamander

蛇鳞 Atom Chain：绑定各段位对应触发器（前段=on_kill/enter_tile，中段=passive，后段=on_hit/segment_lost）
  使用 T25 已有原子 + Build 原子类组合效果

共享的 T25 管线：
  game_config.json
  ├── status_effects     → 状态链（已有，T25）
  ├── reactions          → 反应链（已有，T25）
  ├── snake_heads        → 蛇头链（新增，复用 T25 管线）
  ├── snake_tails        → 蛇尾链（新增，复用 T25 管线）
  └── snake_scales       → 蛇鳞链（新增，复用 T25 管线 + Build 原子）

  所有链 → EffectChainResolver → TriggerManager → AtomExecutor

新增即时原子（4个）：
  modify_food_drop    — 改写击杀后食物掉落数量
  direct_grow         — 直接增长蛇身（跳过食物）
  steal_status        — 从被吃敌人偷取 carried_status
  modify_hit_threshold — 改写掉段所需的攻击次数

EffectWindow 时间窗口框架（T27 基础设施）：
  delay_loss / grant_invincibility / mark_recovery_window 等需要"持续 N tick"的效果
  不继承 AtomBase（原子无状态单例，窗口有生命周期）
  新增 EffectWindowManager（有状态管理器）+ 2 个原子：
    open_window         — 开启时间窗口（duration_ticks / rules / on_expire / cancel_on）
    if_in_window        — 条件原子，查询窗口是否活跃
  窗口期内规则覆写由各系统主动查询（如 ignore_hit_counter / block_segment_loss）
  到期时执行 on_expire 原子链（复用 AtomExecutor），取消时不触发
  完全 JSON 配置，新增窗口型效果零代码
  预留扩展点：on_tick 每tick链（L2+）、窗口叠加策略（refresh/extend/stack）

蛇头/尾等级：
  Ⅰ级（初始）
  Ⅱ级（升级后）：原子参数增强（如 direct_grow amount 1→2）
  Ⅲ级（升级后）：新增原子链（如 Hydra 回响咬 = 新增 echo_bite 原子链）

升级方式：商人间消费蜕皮，或特定事件奖励
替换方式：遗迹间完成解谜；替换后等级清零

交叉组合能力：
  状态原子 × 蛇头原子可互相引用
  例：蛇吃掉带火的敌人 → steal_status → 火的 on_interval 链自动激活在蛇段上
  例：if_has_status 条件原子可作为蛇尾链条件 → "只有带冰状态时才触发恢复"
  蛇鳞 modify_effect_value 可增幅蛇头链中 direct_grow 的 amount
```

### 5.4 蛇头详细设计

#### 九头蛇 Hydra —— 贪婪型
> 改写「吃敌人的收益和代价」：高收益高风险

| 等级 | 规则描述 |
|------|----------|
| Ⅰ | 吃敌人**不掉食物**，直接 **+1 长度** + 蛇头获得被吃敌人的 carried_status。但受击计数器上限 **-1**（2 hit 即掉段，而非标准 3 hit） |
| Ⅱ | 直接 +1 长度改为 **+2 长度**（仍然 2 hit 掉段） |
| Ⅲ | 吃敌人时产生**回响咬**：被吃位置周围 1 格的敌人各受 1 点伤害（仍然 2 hit 掉段） |

**玩家心理：** 不断进攻、不断吃，用长度优势换更多状态段去杀敌。但更脆弱——一旦被打代价翻倍。

**操作特征：**
- 倾向冲进敌群快速吃掉多个
- 长度涨得快 → 更多火光环 / 毒液蔓延覆盖
- 但必须避免被打，否则快速崩盘

---

#### 白蛇 Bái Shé —— 稳健型
> 改写「吃敌人后的安全窗口」：低收益低风险，鼓励防守反击节奏

| 等级 | 规则描述 |
|------|----------|
| Ⅰ | 吃敌人后获得 **3 tick 无敌**（期间受击不计入计数器），但食物掉落数 **-1**（最低 1 个） |
| Ⅱ | 无敌期间被攻击的敌人自动获得 **ice 状态**（防守反击） |
| Ⅲ | 无敌时间延长至 **5 tick**，且无敌结束瞬间对周围 1 格敌人施加蛇头当前 carried_status |

**玩家心理：** 先用状态布局（毒液蔓延铺路、火光环清场），找准时机吃一个敌人获得安全窗口，再调整位置。

**操作特征：**
- 吃一个 → 安全调整位置 → 吃下一个
- 长度涨得慢（食物少），但生存能力强
- Ⅱ级后鼓励"故意在无敌期挡敌人的攻击"来冻住它们

---

**两种蛇头的对比：**

| 维度 | 九头蛇 Hydra | 白蛇 Bái Shé |
|------|-------------|-------------|
| 吃敌人收益 | 直接 +长度（高） | 掉食物 -1（低） |
| 受击容忍 | 2 hit 掉段（脆） | 3 hit + 无敌窗口（硬） |
| 攻击节奏 | 快攻连吃 | 有节奏的安全进食 |
| 典型死因 | 贪心冲太深被围攻 | 太保守没吃够食物被慢慢磨死 |

### 5.5 蛇尾详细设计

#### 再生尾 Salamander —— 防守恢复型
> 改写「丢段后的恢复窗口」：鼓励被打后冷静回避

| 等级 | 规则描述 |
|------|----------|
| Ⅰ | 丢段后开启 **5 秒窗口**；窗口内未再次丢段 → **自动恢复 1 段**（恢复的段继承丢失段的 carried_status）；窗口内再次丢段 → 窗口重置 |
| Ⅱ | 恢复窗口延长至 **7 秒** |
| Ⅲ | 恢复 1 段 + 该段自动获得 **ice 状态**（防御姿态） |

**适配分析：** 与白蛇搭配 → 双重防御（无敌 + 恢复）；与九头蛇搭配 → 弥补脆弱性（2 hit 掉段但能恢复）

---

#### 时滞尾 Lag Tail —— 进攻取消型
> 改写「受击计数器满的结算时机」：鼓励被打了赶紧反咬一口

| 等级 | 规则描述 |
|------|----------|
| Ⅰ | 受击计数器满时，**不立即丢段**，延迟 **3 tick**。延迟期间如果**吃到敌人**，取消本次丢段 |
| Ⅱ | 延迟期间吃到敌人 → 取消丢段 + 受击计数器额外 **-1** |
| Ⅲ | 延迟期间的**任何击杀**（包括火光环、反应伤害）都能取消丢段 |

**适配分析：** 与九头蛇搭配 → 极致进攻（被打了赶紧吃回来）；与白蛇搭配 → 安全网（无敌期间找机会吃掉取消丢段）

## 6. 系统五：蛇鳞系统

> **前置系统：** 系统三（状态效果）、系统四（蛇头/蛇尾）
> **L2 修订版：** 基于 L1 per-segment status 和战斗循环重新设计

### 6.1 系统描述

蛇鳞是「内容层遗物」——**抽象的规则修改器**，装在前/中/后三个段位的槽中。蛇鳞不对应蛇身的具体某一节，而是修改整条蛇或某类行为的规则参数。

### 6.2 设计理念

> 蛇鳞和 per-segment status 是两套独立系统：
> - **per-segment status**（战场层）：每个身体段独立携带 fire/ice/poison，产生火光环/毒液蔓延/冰防御
> - **蛇鳞**（Build 层）：装在抽象槽位中，修改「规则」——比如让火光环伤害+1、毒液蔓延频率+1
>
> 蛇鳞的效果作用于**整条蛇**或**整个系统**，而非具体某一段身体。

### 6.3 设计框架

**槽位结构**

```
[蛇头] — [前段槽A] — [前段槽B] — [中段槽A] — [中段槽B] — [中段槽C] — [后段槽A] — [后段槽B] — [蛇尾]
          （初始开放1个）           （初始开放1个）                       （初始开放1个）

初始槽位：前×1，中×1，后×1（共3槽）
最大槽位：前×2，中×3，后×2（共7槽）
```

**槽位段位的触发逻辑（L1 适配版）**

| 段位 | 触发条件 | 说明 |
|------|----------|------|
| 前段 | 蛇头**吃敌人**或**进入状态格** | 蛇的主动攻击/获取行为 |
| 中段 | **持续被动生效**（装着就运作） | 不变 |
| 后段 | 蛇身**被敌人攻击**或**受击计数器满时丢段** | 蛇的被动受害事件 |

**鳞片等级**

```
Ⅰ级（初始）：基础效果
Ⅱ级：获得同一鳞片第二次时自动升级；效果数值 ×1.5
Ⅲ级：获得同一鳞片第三次时自动升级；效果产生质变（不只是数值，形态改变）
```

**鳞片效果范围分类**

| 范围 | 说明 | 例子 |
|------|------|------|
| 全蛇 | 修改整条蛇的某个规则参数 | "受击计数器上限 +1"、"火光环范围 +1 格" |
| 动作 | 修改某个动作的效果 | "吃敌人额外掉 1 食物"、"毒液蔓延频率 +1" |
| 条件 | 满足条件时触发一次性效果 | "被攻击时击退攻击者"、"丢段时反伤" |

**邻接共鸣**

```
两片相邻槽位中的鳞片，若具有关联标签，触发共鸣词缀
共鸣效果强度 ≤ 两片鳞片效果之和的 70%
共鸣效果可产生超越单片鳞片设计边界的质变效果
共鸣词缀在玩家首次触发时显示提示（发现机制，非教程机制）
```

### 6.4 鳞片分类与设计示例

**标签系统（用于 PCG 权重计算和共鸣判断）**

```

```

## 7. 系统六：敌人系统

> **前置系统：** 系统一、系统二、系统三

### 7.1 系统描述

敌人是在 Grid 上按照固定行为规则移动的实体。分为基础级、战术级、特殊级和精英/Boss 级。敌人不追求「聪明」，而追求「可预测但有威胁的规律」。

### 7.2 设计理念

> 敌人是让玩家 Build「有用武之地」的舞台，不是纯粹的数值对抗。每个敌人对状态格的响应方式（回避/无视/趋向/利用/恐惧）决定了玩家的状态布置是否有意义。
>
> **给关卡策划的指导：** 同一房间内的敌人类型不超过 3 种（认知负担控制）。敌人组合设计应考虑「状态响应类型的多样性」，避免全部使用同一响应类型。

### 7.3 敌人行为框架

**行为优先级栈（每 tick 按顺序检查）**

```
P1  自我保护  →  当前格是危险状态格？→ 移离
P2  威胁响应  →  蛇头在攻击范围内？→ 进入攻击逻辑
P3  状态响应  →  战场上有状态格？→ 按本敌人的响应类型决策
P4  目标追踪  →  向主要目标移动
P5  默认行为  →  无条件触发的兜底行为
```

**状态响应类型（P3）**

| 类型 | 行为描述 |
|------|----------|
| 回避型 | 计算避开所有状态格的路径 |
| 无视型 | 完全忽略状态格 |
| 趋向型 | 主动向特定类型的状态格移动 |
| 利用型 | 主动经过特定状态格以给自己附加有利状态 |
| 恐惧型 | 特定状态格使其停止移动进入僵直 |

### 7.4 敌人分级设计

#### 基础级：游荡者 Wanderer ✅ L1 已实现
```
状态响应：无视型
P0 攻击：attack_cooldown = 3 ticks，attack_range = 1
P4 追踪：无
P5 默认：随机移动，碰壁反弹
HP：1（蛇头一口吞噬）
掉落食物：2
设计意图：战场上的随机触发器，会随机激活状态格
```

#### 基础级：追踪者 Chaser ✅ L1 已实现
```
状态响应：回避型
P0 攻击：attack_cooldown = 2 ticks，attack_range = 1
P4 追踪：向最近蛇身段的相邻空格移动（不追蛇头）
P2 威胁：蛇头进入 3 格范围内，移速 +1
HP：1（蛇头一口吞噬）
掉落食物：3
设计意图：用状态格可以直接引导其路径
```

#### 战术级：毒沼匍匐者 Bog Crawler ✅ L1 已实现
```
状态响应：趋向型（偏好毒液格）
P0 攻击：attack_cooldown = 4 ticks，attack_range = 1
特殊规则：
  踩入毒液格时，移速 +2
  死亡时留下 3 格毒液格
HP：1（蛇头一口吞噬）
掉落食物：4
设计意图：玩家的毒液格是助力还是威胁，取决于当前敌人构成
```

#### 🔮 战术级：分裂体 Splitter（L2+ 未实现）
```
状态响应：恐惧型（冰霜格）
特殊规则：
  距蛇头 1 格时，读条 1 秒后分裂为 2 只小分裂体（HP=1，随机移动）
  灼烧状态加快读条速度至 0.5 秒
  冰冻状态打断分裂读条
HP：2 格（小分裂体 1 格）
设计意图：灼烧反而催促分裂；冰霜格是克制手段
```

#### 🔮 特殊级：战场读取者 Sentinel（L2+ 未实现）
```
状态响应：利用型（最复杂）
每 tick 执行战场扫描：
  计算「经过 X 格，将与自身状态触发 Z 反应」
  选择触发最高伤害反应的移动路径
特殊规则：
  自身携带随机初始状态（每次生成时随机）
  击杀时在死亡位置触发该状态与当前地板状态的反应
HP：4~6 格
设计意图：会反利用玩家铺设的状态格；高难度标志性敌人
```

### 7.5 🔮 部位破坏系统（精英/Boss 专用，L2+ 未实现）

大型敌人身体分为若干部位，每个部位独立计算破坏进度。

```
部位数据结构：
  id          : 部位标识
  hp          : 需要被攻击的次数（折算为长度消耗）
  type        : 部位类型（功能型/危险型/奖励型）
  state       : 当前状态（完好/受损/摧毁）
  effect_on_destroy : 摧毁时触发的效果

击杀判定：
  蛇头撞击某部位 → 该部位 hp -1 → hp 归零时「摧毁」
  所有「核心部位」被摧毁 → 敌人死亡
```

**部位类型**

| 类型 | 摧毁后效果 |
|------|-----------|
| 功能型（眼睛） | 敌人失去追踪，变为随机移动 |
| 功能型（护甲节） | 该区域攻击消耗恢复正常（护甲存在时消耗 ×2） |
| 危险型（毒囊） | 爆裂，原位留下毒雾区域（玩家也受影响） |
| 危险型（能量核） | 爆炸，范围内所有实体击退 3 格 |
| 奖励型（宝囊） | 掉落蜕皮 ×3 |
| 奖励型（蜕皮节） | 掉落随机鳞片 |
| 核心部位 | 全部摧毁 → 敌人死亡 |

**精英示例：蛇蜕魁**

```
部位：
  旧皮护甲 ×3（功能型，摧毁前撞击消耗 ×2）
  毒囊 ×2（危险型，摧毁后留毒雾）
  蜕皮核（核心，需护甲全部摧毁后才暴露，攻击 3 次击败，掉落鳞片）

阶段切换：
  护甲全部摧毁 → 移速 +1，状态响应从「回避型」切换为「恐惧型（冰霜格）」
  毒囊被摧毁时 → 追加每 5 秒在当前位置生成毒液格
```

### 7.6 敌人 AI 可读性设计（视觉需求）

```
敌人头顶状态图标：
  追踪中  → 红色箭头指向目标
  回避中  → 黄色弯箭头
  读条中  → 进度条
  恐惧中  → 紫色感叹号

移动预判：追踪型敌人下一步目标格有淡色高亮（仅 1 格）
状态响应时：敌人与目标状态格之间短暂连线提示
```

## 8. 系统七：地图与房间系统（PCG）

> **前置系统：** 系统三、系统六

### 8.1 系统描述

地图由多个房间组成，房间之间通过通道连接，玩家可在同一层内反复进出已探索的房间。层级通过 PCG 规则组合生成，保证每局 Run 的层序列不同但始终可玩。

### 8.2 设计理念

> PCG 不是「随机生成所有东西」，而是「从有限的规则组合中生成有意义的排列」。随机的部分是「选什么、怎么组合」，固定的部分是「每个元素本身的规则和层的基本结构」。
>
> **给关卡策划的指导：** 设计新的地形模板、机制修饰、敌人组合时，需要同时考虑「这个元素和哪些状态效果/Build 方向会产生有趣的化学反应」。所有内容单元都应具备独立的「设计意图标注」。

### 8.3 层级结构

```
一层 = 中心广场 ×1 + 普通房间 ×4~6 + 精英房间 ×1 + Boss 房间 ×1

中心广场：
  无敌人，无机制
  食物缓慢刷新
  玩家在此可开启鳞片排布界面
  是各房间之间的缓冲区

Boss 房间：
  只有 1 条入口
  击败 Boss → 解锁进层
```

**层内时间压力：腐化进度**

```
进入一层开始计时，腐化进度持续推进：
  腐化1 → 地图边缘格子开始消失
  腐化2 → 中心广场食物停止刷新
  腐化3 → 已清空的房间重新生成敌人（更强版本）
  腐化4 → Boss 房间强制开启，无法继续拖延

腐化进度对玩家可见（进度条）
```

**房间状态（反复探索机制）**

```
首次进入  → 完整内容（敌人/食物/机制全部存在）
再次进入  → 「残局状态」：已击杀敌人不复活；食物重新刷新（量减半）；地形机制依然有效
三次进入  → 「枯竭状态」：无敌人，无食物；可能暴露首次进入时未发现的隐藏格
```

### 8.4 PCG 生成流程

**Step 1：抽取层主题（两个标签交叉）**

```
环境标签（各自独立的内容池，关卡策划填充）：
  地下洞穴 / 腐化沼泽 / 古代遗迹 / 机械内部 / 虚空裂缝

压力标签：
  猎场（敌人密度高，食物少）
  荒原（食物多，敌人少但强）
  迷宫（连通复杂，隐藏内容多）
  竞速（腐化速度加快）
  围困（房间出口在完成目标前锁定）
```

**Step 2：生成房间池**

```
基础配额（每层固定）：战斗间 ×2，中心广场 ×1，Boss 间 ×1
主题追加（按压力标签权重）：
  迷宫标签  → 隐藏间概率 +，遗迹间概率 +
  猎场标签  → 精英间概率 +，战斗间数量 +
  荒原标签  → 猎食间概率 +，商人间概率 +
  竞速标签  → 所有房间追加「腐化加速」修饰
```

**Step 3：房间内容生成（三层叠加）**

```
底层：地形模板（物理结构）
  开阔型   → 大量空地，少量障碍
  走廊型   → 多条窄通道
  岛屿型   → 障碍物分隔多区域
  螺旋型   → 天然引导蛇做螺旋运动
  非对称型 → 一半开阔一半密集

中层：机制修饰（0~2 个，从修饰池抽取，同层不重复）
  地形类：地雷格 / 传送门对 / 腐化地板
  规则类：黑暗（视野缩减）/ 加速带 / 镜像轴（左右操作反转）
  敌人类：增援 / 护盾 / 联动（敌人死亡时触发相邻敌人狂暴）

表层：内容填充
  敌人：不在出入口附近生成；单房间敌人类型 ≤ 3 种
  食物：与敌人保持最小距离；特殊食物生成于非显眼位置
```

**Step 4：连通结构生成**

```
固定锚点：中心广场居中，Boss 间在最远端（单入口）
动态连接（三种形态）：
  星形  → 所有房间直连中心广场
  链式  → 房间串联，中心广场在中间，有主路/支路之分
  网状  → 房间交叉连接（迷宫标签时权重最高）
保证每个房间至少 2 条通路（避免死胡同）
```

**层间 PCG 关联（后一层参考前一层）**

```
参考玩家当前蛇头/尾标签 → 避免连续两层出现克制同一标签的地脉
参考玩家鳞片构成       → 若全为[毒]标签，下一层「毒抗敌人」出现权重 +
参考玩家通关方式       → 主要靠战斗还是探索，影响下一层战斗间/猎食间比例
```

### 8.5 房间类型详细设计

| 房间类型 | 目标 | 奖励 | 备注 |
|----------|------|------|------|
| 战斗间 | 消灭所有敌人 | 鳞片 ×1（3 选 1） | 失败不强制死亡，可撤退 |
| 猎食间 | 限时内吃够 N 个食物 | 长度永久 +2 | 无敌人，但食物在移动 |
| 遗迹间 | 抵达特定格子（有机关） | 蛇头/尾替换选项 | 每层最多出现 1 次 |
| 商人间 | 消费蜕皮货币 | 自选鳞片/升级/槽位 | 无战斗压力 |
| 诅咒间 | 强制进入，无法跳过 | 通过即离开 | 附加诅咒鳞+补偿 |
| 精英间 | 击杀精英怪 | 鳞片 ×2 + 特殊事件 | 精英有专属部位机制 |
| 隐藏间 | 探索（无明确目标） | 隐藏格/鳞片碎片 | 枯竭状态才出现隐藏内容 |

### 8.6 房间选择界面需求

```
战斗/精英间完成后，显示 2~3 个候选下一房间
每个候选房间显示：房间类型、地形机制标注、敌人数量概述、奖励类型
玩家做出选择后进入
```

## 9. 系统八：事件系统

> **前置系统：** 系统五（蛇鳞）、系统四（蛇头/蛇尾）、系统七（地图）

### 9.1 系统描述

事件系统是 Build 的「意外转折点」。事件不是系统主动插入的奖励，而是玩家在世界里发现的客观存在物，玩家自主决定是否互动。分为三种来源：建筑物、掉落物、痕迹。

### 9.2 设计理念

> 事件是「世界里本来就存在的东西」，不是「系统找借口给玩家东西」。玩家不互动，事件依然存在于地图中。互动的决策完全由玩家主导。
>
> **给叙事设计的指导（待填充）：** 所有事件的文本描述需要简洁（1~2 句话），文本是氛围服务，不是信息载体。机制信息通过 UI 元素呈现，不依赖文本理解。

### 9.3 来源 A：事件建筑物

建筑物作为地形元素在房间生成时嵌入，玩家蛇头经过相邻格时触发互动提示，游戏在此暂停。

**建筑物示例：蜕皮祭坛**

```
外观需求：刻有蛇纹的石台，有空槽（待美术）
触发：蛇头经过相邻格

互动：玩家将一片鳞片放入石台（从当前槽位移除）→ 孵化

孵化结果（按放入鳞片的标签决定，关卡策划填充孵化表）：
  示例：[火]鳞片 → 孵化出「熔岩纹」（扩大版火焰鳞，但触发时伤害自身 1 格）

不互动：建筑物一直存在，不影响地图功能
```

**建筑物示例：古蛇遗骸**

```
外观需求：巨大石化蛇骨横亘房间，本身是地形障碍（待美术）
触发：蛇头碰触遗骸某个部位，该部位亮起

解谜机制：遗骸有头/身/尾三个部位，按顺序碰触触发不同效果
  头→身→尾（顺序）→ 「古蛇传承」：蛇头升一级
  尾→身→头（逆序）→ 「逆鳞之道」：蛇尾升一级，蛇头降一级
  随机顺序    → 「惊扰亡灵」：遗骸碎裂成永久障碍，各碎片位置留下鳞片碎片

不互动：遗骸作为地形障碍影响走位，本身具有地图设计价值
```

### 9.4 来源 B：掉落物

精英怪有概率掉落「遗物碎片」。碎片不占鳞片槽位，悬挂在蛇头上方（视觉标识）。持有时世界对玩家产生不同反应，需主动激活才能获得最大价值。

**掉落物示例：破碎的眼球**

```
掉落来源：镜像蛇精英
持有效果：所有敌人头顶显示下一步移动方向（仅 1 格）

激活方式 A：带碎片进入商人间 → 商人提供特殊交易
  用破碎眼球换「完整眼球鳞片」
  完整眼球鳞片效果：前段触发时，范围内所有敌人被「标记」，标记的敌人受到的下一次攻击消耗减半

激活方式 B：带碎片击杀另一只镜像蛇 → 两个碎片合并
  合并为「双眼鳞片」
  效果：蛇头和蛇尾同时获得行为预判显示

不激活：层结束时自动消失（不跨层保留）
```

**掉落物示例：毒囊残渣**

```
掉落来源：蛇蜕魁精英（摧毁毒囊部位时掉落）
持有效果：蛇身经过的格子，毒液格生成概率 +50%

激活方式 A（解谜路线）：找到本层的「净化泉」建筑物，将残渣放入
  → 净化为「蒸馏毒液鳞」：毒液格中毒效果持续时间 ×2

激活方式 B（战斗路线）：持有残渣时，主动走入火焰格
  → 残渣被引燃，触发毒爆反应
  → 爆炸范围内所有格子变为增强毒液格（持续时间 ×3）
  → 残渣消失
```

### 9.5 来源 C：痕迹

地图上自然存在的环境标记，不主动提示，需要玩家自己注意到。蛇头经过时出现极简提示，玩家选择是否停留互动。

**痕迹示例：旧日蛇迹**

```
外观需求：地板上消退的蛇形轨迹纹路（待美术）
经过提示：一行文字（叙事待填充）

停留互动（操作解谜）：
  蛇头沿旧日蛇迹的路径精确移动（不能偏离超过 2 格）
  完整走完轨迹 → 触发「共鸣」：获得旧日那条蛇遗留的一片鳞片碎片

走偏或放弃：旧日蛇迹消失，无奖励
不互动：痕迹一直存在，可以回头互动
```

**痕迹示例：食物圆阵**

```
外观：五个食物排列成完美的圆形（无额外提示）

直接吃：+5 格长度（正常操作）

隐藏互动（无文字提示，玩家自行发现）：
  用蛇身完整围住圆阵一圈而不破坏任何食物
  → 五个食物同时被吸收，+5 格长度
  → 额外触发「圆满」效果：当前所有邻接共鸣效果激活一次

设计意图：对不了解的玩家只是 5 个食物；对了解的玩家是高价值操作挑战
```

### 9.6 事件感知规则（PCG 权重调整）

```
感知玩家状态（影响事件出现概率）：
  持有 3 片同标签鳞片时  → 轮盘类事件权重 +
  槽位有空位时          → 记忆市场类事件权重 +
  本层战斗次数少        → 掉落物类事件权重 +
  本层连续受伤          → 食物类痕迹权重 +

感知层主题：
  腐化沼泽层 → 净化泉建筑物权重 +
  迷宫标签层 → 旧日蛇迹权重 +，古蛇遗骸权重 +
```

## 10. 系统九：成长与奖励系统

> **前置系统：** 系统四（蛇头/蛇尾）、系统五（蛇鳞）、系统七（地图）

### 10.1 系统描述

成长系统管理一局 Run 内玩家 Build 的扩展和深化。分为四个独立维度：长度（流动资源）、鳞片（内容成长）、槽位（容量扩展）、蛇头/尾（规则深化）。

### 10.2 设计框架

**四个成长维度的稀缺性梯度**

```
鳞片成长   → 常见，每个战斗间/精英间都有产出
槽位扩展   → 稀缺，主要来自 Boss 奖励，每层最多 1 次
蛇头/尾深化 → 罕见，升级来自商人间，替换来自遗迹间
```

**流通货币：蜕皮**

| 来源 | 获得量 |
|------|--------|
| 击杀普通敌人 | +1 |
| 击杀精英 | +3 |
| 放弃鳞片选项 | +2 |
| 枯竭房间隐藏格 | +1~2 |

| 消耗 | 花费 |
|------|------|
| 商人间购买鳞片 | -2~4（按稀有度） |
| 商人间解锁槽位 | -5 |
| 商人间升级蛇头/尾 | -4 |
| 重排槽位顺序 | -1 |

> **数值策划注意：** 单层正常游玩积累约 6~10 蜕皮，商人间最大消费约 10~12 蜕皮。每层勉强够用，需要取舍。蜕皮不跨层清零，但下层商人物价略有上涨。

### 10.3 奖励来源与产出对照

| 奖励来源 | 产出内容 | 频率 |
|----------|----------|------|
| 战斗间清怪 | 鳞片 ×1（3 选 1） | 每个战斗间一次 |
| 猎食间完成 | 长度永久 +2 | 每个猎食间一次 |
| 精英间击败 | 鳞片 ×1（2 选 1，质量更高）+ 蜕皮 ×3 | 稀缺 |
| 商人间 | 自选：鳞片/槽位/蛇头尾升级 | 消费蜕皮，主动获取 |
| 遗迹间完成 | 蛇头/尾替换 | 每层最多一次 |
| 隐藏格触发 | 随机鳞片碎片 | 探索奖励，不可预期 |
| Boss 击败 | 槽位解锁（固定）+ 进层奖励 3 选 1 | 每层一次 |

### 10.4 鳞片成长轴

**等级升级**（同一鳞片获得多次自动升级，见系统五）

**进层奖励 3 选 1（每次从三类各抽一个）**

```
扩展类（加新内容）：
  随机获得 1 片高级鳞片
  获得一片「诅咒鳞」+ 强力补偿鳞片

强化类（深化现有）：
  当前最低等级鳞片免费升一级
  蛇头或蛇尾升级（二选一）

修正类（调整方向）：
  免费重排一次所有鳞片槽位顺序
  将一片鳞片替换为同标签的随机鳞片
```

> 保证进层奖励永远有至少一个选项在强化已有内容，不出现三个选项全部无关的情况。

### 10.5 槽位成长路径

```
开局         前×1  中×1  后×1（共 3 槽）
第 1 层 Boss → 新增 1 槽（玩家选择加在哪个段：前/中/后）
第 2 层 Boss → 同上
商人间（稀缺）→ 花费 5 蜕皮，新增 1 中段槽
第 3 层 Boss → 新增 1 槽，同时解锁「槽位共鸣」功能（邻接共鸣激活条件从「相邻」放宽）
上限         前×2  中×3  后×2（共 7 槽）
```

### 10.6 成长节奏总览

| 层数 | 成长重点 |
|------|----------|
| 第 1 层 | 填满基础 3 槽；确立 Build 雏形 |
| 第 2 层 | 第一次槽位扩展；鳞片开始出现 Ⅱ 级；可考虑替换蛇头/尾 |
| 第 3 层 | 邻接共鸣开始有意义；槽位接近上限；Build 方向基本锁定 |
| 第 4 层+ | 鳞片冲 Ⅲ 级质变；蛇头/尾升至 Ⅱ~Ⅲ 级；Build 开始「自运转」 |
| 终 Boss | 验收整局 Build |

## 11. 系统十：数值框架与难度曲线

> **前置系统：** 所有系统

### 11.1 基础数值基准（第一层，普通状态）

**玩家基准（L1 修订）**

| 项目 | 数值 |
|------|------|
| 初始长度 | 6 格 |
| 吃敌人消耗 | 0 格（直接吞噬） |
| 每击杀敌人掉落食物 | 2~4 个（按敌人类型） |
| 单次食物补充 | +1 格 |
| 敌人攻击蛇身 | 每 3 次命中丢 1 段（hits_per_segment_loss=3） |
| 场上最大敌人数 | 3 |
| 场上最大食物数 | 3 |

**敌人基准（L1 第一层）**

| 敌人类型 | HP | 攻击冷却 | 掉落食物 |
|----------|----|----------|----------|
| wanderer | 1 | 3 ticks | 2 |
| chaser | 1 | 2 ticks | 3 |
| bog_crawler | 1 | 4 ticks | 4 |

> **L1 变更：** 所有敌人 HP=1（蛇头一口吞噬）。战斗消耗不再是"吃敌人消耗长度"，而是"敌人攻击蛇身累积受击"。精英/Boss 部位系统待 L2+ 实现。

### 11.2 长度安全阈值

| 区间 | 长度 | 游戏感受 |
|------|------|----------|
| 危险区 | 1~4 格 | 任何失误都可能致命 |
| 生存区 | 5~10 格 | 基本安全，可打普通小怪 |
| 战斗区 | 11~20 格 | 可主动进攻精英 |
| 舒适区 | 21~35 格 | 操作压力低 |
| 过载区 | 36 格+ | 空间压力超过战斗压力 |

### 11.3 层级数值缩放（L2+ 待设计）

> **L1 修订注意：** L1 战斗模型改为「吃敌人无消耗 + 敌人攻击蛇身」，以下"消耗"概念不再适用。L2+ 实现多层系统时需按新模型重新设计缩放曲线。以下保留为参考框架。

| | 第 1 层 | 第 2 层 | 第 3 层 | 第 4 层 | 第 5 层 |
|--|--|--|--|--|--|
| 敌人攻击频率 | 低 | 低 | 中 | 中 | 高 |
| 精英 HP（部位数） | 3 | 4 | 5 | 6 | 8 |
| Boss HP（部位数） | 6 | 8 | 10 | 12 | 15 |
| 食物补充 | 1 格 | 1 格 | 1 格 | 1 格 | 1 格 |
| 击杀掉落食物 | 2~4 | 2~4 | 2~3 | 2~3 | 1~3 |

**关键设计：** 食物补充量不变，敌人攻击频率和部位数递增。难度由空间压力和攻击密度制造。

### 11.4 难度的三个维度

```
维度一：战斗烈度（消耗压力）
  敌人部位增多，护甲/特殊机制更频繁
  第 1~2 层缓慢增长

维度二：资源密度（补给压力）
  食物总量减少，食物位置更危险
  第 3~4 层开始施压

维度三：环境复杂度（认知压力）
  房间修饰数量增加，状态反应链更长，敌人 AI 组合更复杂
  第 5 层+ 全面介入
```

### 11.5 动态难度调整（隐性）

PCG 在生成下一房间时，根据玩家当前状态微调（对玩家不可见）：

```
玩家长度远超预期（过于强势）：
  下一房间敌人护甲概率 +20%；食物量 -1~2；追加 1 个战术级敌人

玩家长度远低于预期（过于弱势）：
  下一房间普通小怪减少 1 只；食物量 +2~3；移除 1 个修饰

玩家状态反应触发次数为 0（未使用状态系统）：
  下一房间追加「状态格已预置」修饰，引导接触状态效果
```

### 11.6 无尽模式难度滚雪球

```
第 1~5 层    正常节奏，Build 积累期
第 6~10 层   食物量每层 -1 个；敌人消耗 +1 格
第 11~15 层  引入「诅咒层」：每 5 层强制叠加 1 个全局负面规则（可叠加，不上限）

诅咒层示例：
  诅咒①：所有食物补充变为 0.5 格（吃两个才补 1 格）
  诅咒②：敌人死亡后留下随机状态格
  诅咒③：每次进入新房间，随机 1 片鳞片槽位暂时封印

第 20 层+   「熵增」机制：每隔 3 层地图可用格子永久减少一圈
```

### 11.7 状态效果数值规则

**状态效果总伤害设计原则**：烈度 × 时间 = 固定总值

| 状态 | 总伤害折算 |
|------|-----------|
| 灼烧 | -3 格（每 2 秒 -1 格，持续 6 秒） |
| 中毒 | ~-2 格效率损失（非直接数值） |
| 冰冻 | 0 格（纯操控惩罚） |
| 腐蚀 | 0 格（纯 Build 惩罚） |
| 感电 | -N 格（取决于接触次数） |
| 虚化 | 0 格（纯效果，无数值损失） |

**反应伤害公式**

```
反应伤害（格）= (状态A层数 + 状态B层数) × 基础系数

系数参考：
  低伤害反应（蒸腾、融化）：0.5
  中伤害反应（毒爆、麻痹）：1.0
  高伤害反应（超导、腐化）：1.5

平衡参考上限：单次反应不超过 4 格长度损失
```

## 12. 系统十一：元成长系统

> **前置系统：** 所有系统（元成长依赖全部系统稳定后设计）

### 12.1 系统描述

元成长分为两个独立层次：「见识」（横向扩展内容池）和「传承」（有限度影响下一局的倾向）。元成长不提供数值补偿，不让玩家变强，只让每局 Run 的可能性变多，以及在局与局之间保持叙事连续性。

### 12.2 设计理念

> 每局 Run 的起跑线在机制上始终平等。元成长影响的是「倾向」，不是「结果」。通关永远需要本局的真实表现。
>
> **给关卡策划的指导：** 设计蛇头/尾的解锁条件时，条件本身应当是对对应机制深度的自然探索。解锁到某个蛇头，意味着玩家已经理解了该蛇头的核心机制。

### 12.3 第一层：见识系统

**蛇头/蛇尾解锁**（通过「做到某件事」解锁，不靠货币购买）

| 内容 | 解锁条件 | 设计逻辑 |
|------|----------|----------|
| 彩虹蛇 Ungud | 单局 Run 累计转向超过 200 次 | 已足够熟悉「转向作为工具」 |
| 美杜莎 Medusa | 单局 Run 用状态反应击杀超过 15 只敌人 | 已发现「状态联动」 |
| 饕餮 Taotie | 单局 Run 在长度低于 5 格的状态下存活超过 60 秒 | 已理解「极限求生」的感觉 |
| 冥尾 Styx Tail | 单局 Run 触发特定「死里逃生」事件后继续通关 | 已经历过死亡边缘 |

**鳞片解锁**

```
鳞片不靠特殊条件解锁
「见过」即解锁：在某局 Run 中触碰到某片鳞片（即使未选择）→ 永久进入内容池
早期 Run 鳞片池小，随探索扩大；「我以前没见过这片」本身是探索乐趣
```

### 12.4 第二层：传承系统

**核心概念：遗愿**

每局 Run 结束时，系统生成一条「遗愿」：

```
生成规则：
  取本局 Run 中「最高光的时刻」
  最高光 = 单次触发效果最强 / 最长存活时间 / 最复杂的状态反应链

遗愿示例：
  「那条蛇曾经以 3 格长度击败了 Boss」
  「那条蛇曾经触发了超过 5 层的状态反应链」
```

**遗愿的继承**

```
下局开局时，玩家可选择继承一条遗愿（可选，非强制）
继承不复现那个时刻，只获得一个「起点倾向」：

继承「超导反应遗愿」→ 开局鳞片池中[冰][电]标签鳞片出现概率 +20%
继承「极限存活遗愿」→ 开局蛇尾候选中「冥尾」出现概率 +50%

继承规则：
  每局只能继承一条
  继承后该遗愿消失（被用掉）
  石碑最多储存 5 条遗愿；超过 5 条时最古老的自动消失
```

**遗愿的叙事价值**

```
石碑上的 5 条遗愿是玩家自己的游玩历史
每条遗愿都是真实发生过的故事
继承时带着那个记忆开始新的一局
元成长从「数值积累」变为「故事积累」
```

### 12.5 无尽模式元成长变体

```
无尽模式每隔 10 层触发「蜕皮时刻」（代替普通死亡触发遗愿）：
  游戏暂停
  玩家必须放弃当前一片鳞片（蜕掉旧皮）
  放弃的鳞片变成「无尽遗愿」，记录在独立的无尽石碑上
  同时获得一片新鳞片（3 选 1）

无尽石碑满 10 条时，解锁一个只在无尽模式出现的特殊蛇头
```

### 12.6 永不跨局传承的内容

```
长度（每局从 6 格开始）
蜕皮货币（每局从 0 开始）
鳞片槽位（每局从 3 槽开始）
具体的鳞片配置（每局重新 Build）
```

## 12A. L1 实现修订（Gameplay Loop Redesign）

> **适用范围：** L1 Gameplay Loop 实现阶段（基础移动 + 长度 + 状态效果 + 敌人）
> **修订原因：** 原设计核心循环「长度 = 攻击消耗」在 L1 原型验证中调整为「吃敌人 + 蛇身受击」模型

### 12A.1 设计理念变更

| 维度 | 原设计 (v0.1) | L1 修订 |
|------|---------------|---------|
| 核心循环 | 长度 = 唯一数值单位，攻击消耗长度 | 吃敌人无成本获得食物，敌人主动攻击蛇身 |
| 风险来源 | 长度消耗的机会成本 | 暴露蛇身给敌人的空间风险 |
| 回报来源 | 击杀后获取资源补偿 | 击杀后的食物爆发（位置散落） |
| 蛇身角色 | 生命值 + 攻击资源 | 状态载体 + 被攻击面 |

### 12A.2 蛇头吃敌人

蛇头碰撞敌人 = 直接吞噬（不消耗长度），敌人死亡后掉落食物。**所有击杀方式均掉落食物**（蛇头吞噬、火光环、反应伤害等），统一在 `enemy_killed` 信号中处理。

**吞噬状态反应**：若蛇头携带状态 A、敌人携带状态 B 且 A≠B，则在吞噬前触发 reaction（蒸腾/毒爆/冻疫），蛇头状态清除；同类状态则无额外效果。

```
敌人击杀 → 食物掉落配置：
  wanderer:    drop_food_count = 2
  chaser:      drop_food_count = 3
  bog_crawler: drop_food_count = 4
```

食物散落规则：在击杀位置周围的空格中随机放置。

### 12A.3 蛇身段独立携带状态（Per-Segment Status）

每个 `SnakeSegment` 拥有独立状态字段：

```
SnakeSegment:
  carried_status: String  # "fire" | "ice" | "poison" | ""
```

- 蛇不再使用 `StatusEffectManager` 管理自身状态（该系统仅用于敌人）
- 状态可视化直接绑定在每个身体段上（覆盖层 + 边框）
- **段对象持久化**：蛇移动时所有段对象向前移动一格（不创建/销毁），carried_status 自然跟随段走
- 蛇基础颜色为白/灰（HEAD=0.95, BODY=0.78, TAIL=0.6 灰度），与毒绿色区分

### 12A.4 敌人攻击蛇身（Enemy Body Attack）

敌人 AI 新增 **P0 优先级**：攻击判定（高于所有移动行为）。

| 参数 | 说明 |
|------|------|
| `attack_range` | 攻击范围，默认 1 格（曼哈顿距离） |
| 攻击目标 | 非 HEAD 的蛇身段 |
| `attack_cooldown` | 攻击冷却（单位：tick） |

```
攻击冷却配置：
  wanderer:    attack_cooldown = 3 ticks
  chaser:      attack_cooldown = 2 ticks
  bog_crawler: attack_cooldown = 4 ticks
```

**双向状态转移**：敌人攻击蛇段时双方状态互相传递：

| 敌人状态 | 蛇段状态 | 结果 |
|----------|----------|------|
| 无 | 有状态 | 敌人沾染蛇段状态（视觉同步更新） |
| 有状态 | 无 | 蛇段获得敌人状态 |
| 有状态 A | 有不同状态 B | 触发反应，双方状态清除 |
| 有状态 | 相同状态 | 无事发生 |

敌人携带状态时显示对应的覆盖层颜色（火=红橙边框闪烁、冰=蓝色覆盖、毒=绿色脉动），与蛇段视觉一致。

### 12A.5 受击计数器（Hit Counter System）

蛇维护全局受击计数器，避免单次攻击秒杀：

```
Snake:
  hits_taken: int = 0
  hits_per_segment_loss: int = 3  # 默认值

受击流程：
  敌人攻击命中 → hits_taken += 1
  当 hits_taken >= hits_per_segment_loss:
    → 失去 1 段身体（尾部）
    → hits_taken 重置为 0
```

### 12A.6 状态格交互重写（Status Tile Interaction）

每个蛇身段独立与脚下状态格交互，采用统一传播规则：

| 蛇段状态 | 格子状态 | 结果 |
|----------|----------|------|
| 无（""） | 有状态 | 段获得格子的状态 |
| 有状态 | 相同状态 | 无事发生 |
| 有状态 | 不同状态 | 触发反应 + 清除段上状态 + 消除该格子 |

- 状态格永久存在（无持续时间递减）
- 与原设计 4.1~4.3 的「层数」「持续时间」机制不同，L1 简化为二值状态
- **同位置异类型状态格互斥**：放置新格子时如已有不同类型，触发反应 + 双方消除（不放置新格子）
- 反应同时消除触发位置的状态格（不再保留）

### 12A.7 蛇身段增益效果（Segment Effect System）

携带状态的蛇身段产生被动效果：

| 状态 | 效果名称 | 描述 |
|------|----------|------|
| fire | 火焰光环 | 每 tick 对相邻格（曼哈顿距离 1）的敌人造成**火属性**伤害；若目标敌人携带异类状态则触发反应、双方状态清除 |
| poison | 毒液蔓延 | 每个携带 poison 的蛇段每 3 tick 向相邻 4 格中随机一个无蛇身的空格蔓延一格毒状态格（间隔可配置）；蔓延到异类状态格时自动触发反应 |
| ice | 冰霜防御 | 携带 ice 的段被攻击时，攻击者获得 ice 状态（已合并入双向状态转移） |

### 12A.8 跨类型反应（L1 仅 3 种）

L1 实现从原设计 15 种反应中精简为 3 种：

| 反应名称 | 触发条件 | 效果 |
|----------|----------|------|
| 蒸腾（steam） | fire + ice | 范围伤害 2 + 蛇自伤 1 |
| 毒爆（toxic_explosion） | fire + poison | 范围伤害 3 + 蛇自伤 2 |
| 冻疫（frozen_plague） | ice + poison | 范围内敌人施加 ice + poison（无直接伤害） |

### 12A.9 无身体倒计时（No-Body Countdown）

蛇失去所有身体段（仅剩蛇头）时不会立即死亡，进入倒计时阶段：

```
无身体倒计时：
  触发条件：body_segments.size() == 0
  持续时间：10 秒 = 40 ticks（默认）
  取消条件：吃到食物恢复身体段
  倒计时归零 → 死亡

可视化：
  蛇头加速闪烁（黄 → 红）
  HUD 显示 "EAT OR DIE: Xs"
```

### 12A.10 敌人数值调整

| 敌人类型 | 原设计 HP | L1 HP | 攻击冷却 | 击杀掉落食物 |
|----------|-----------|-------|----------|--------------|
| wanderer | — | 1 | 3 ticks | 2 |
| chaser | — | 1 | 2 ticks | 3 |
| bog_crawler | 2 | 1 | 4 ticks | 4 |

所有敌人 HP 统一为 1（蛇头一口吞噬），通过攻击冷却和移动模式区分威胁度。

## 12B. L2 前置基础设施：EffectWindow 时间窗口框架（T27）

L2 蛇头/蛇尾系统需要"持续 N tick 的效果窗口"能力。此节记录 T27 的设计决策。

### 12B.1 问题与动机

T25 Atom System 的原子是**无状态即时执行**（AtomBase.execute() 调一次完成）。L2 蛇头/蛇尾需要有生命周期的效果：

| L2 需求 | 窗口行为 |
|---------|---------|
| 白蛇吃后无敌 | 开 invincible 窗口 3 tick → 期间受击不计数 → 到期释放状态 |
| 时滞尾延迟丢段 | 开 delay_loss 窗口 3 tick → 期间阻止丢段 → 到期执行丢段 → 击杀取消 |
| 再生尾恢复窗口 | 开 recovery 窗口 20 tick → 到期自动 +1 段 → 再次受击取消 |

共同模式：**开窗口 → 存续期间改规则 → 到期/取消执行不同逻辑**。

### 12B.2 架构决策

**不继承 AtomBase**。原因：
- AtomBase 实例是单例（注册在 atom_registry 中共享），不能持有状态
- 窗口是有生命周期的对象（创建→存续→销毁），属于不同抽象层
- 分层更清晰：原子负责"开窗口"这个动作，窗口管理器负责生命周期

**分层关系：**
```
AtomBase（不变，无状态即时执行器）
    ↓ open_window 原子的 execute() 调用
EffectWindowManager（新增，有状态管理器，挂在 GameWorld 上）
    ↓ 管理 N 个并存窗口实例
EffectWindow（新增，RefCounted 数据对象）
    ↓ tick 递减 / 到期回调 / 条件取消
```

### 12B.3 EffectWindow 数据模型

| 字段 | 类型 | 说明 |
|------|------|------|
| window_id | String | 唯一标识（"invincible"、"delay_loss"、"recovery"） |
| owner | Object | 窗口持有者（Snake 实例） |
| remaining_ticks | int | 剩余 tick 数 |
| rules | Dictionary | 窗口期内规则覆写（被动数据，各系统主动查询） |
| on_expire_defs | Array | 到期时执行的原子链定义（JSON Array） |
| on_tick_defs | Array | [L2+ 预留] 每 tick 执行的原子链定义 |
| cancel_on | String | 取消触发器 ID（复用已有触发器名，如 "on_kill"） |

### 12B.4 新增原子

| 原子 | 类型 | JSON 参数 | 作用 |
|------|------|-----------|------|
| open_window | 动作 | window_id, duration_ticks, rules, on_expire, on_tick, cancel_on | 开启/刷新时间窗口 |
| if_in_window | 条件 | window_id | 查询指定窗口是否活跃 |

### 12B.5 JSON 配置模式

```json
{
  "atom": "open_window",
  "window_id": "invincible",
  "duration_ticks": 3,
  "rules": { "ignore_hit_counter": true },
  "on_expire": [
    { "atom": "burst_status", "range": 1 }
  ],
  "on_tick": [],
  "cancel_on": ""
}
```

**JSON 驱动能力：** 新增窗口型效果只需配置 JSON，无需编写 GDScript。

### 12B.6 规则覆写消费方

窗口的 `rules` 是被动数据，由各系统主动查询 `EffectWindowManager.get_rule()`：

| rule key | 消费者 | 效果 |
|----------|--------|------|
| ignore_hit_counter | enemy.gd:_attack_segment | 受击不计入计数器 |
| block_segment_loss | snake.gd:remove_tail_segment | 阻止实际丢段 |

### 12B.7 生命周期

1. **开启**：`open_window` 原子执行 → EffectWindowManager 创建/刷新窗口 → 发射 `window_opened` 信号
2. **存续**：每 tick 递减 remaining_ticks；各系统查询 rules 获取覆写值
3. **到期**：remaining_ticks ≤ 0 → 执行 on_expire 原子链 → 移除窗口 → 发射 `window_expired` 信号
4. **取消**：cancel_on 对应的事件触发 → 移除窗口（不执行 on_expire）→ 发射 `window_cancelled` 信号
5. **刷新**：同 id 重复开窗口 → 刷新 remaining_ticks（不叠加，不重新触发 on_expire）

### 12B.8 L2+ 预留扩展点

| 扩展点 | 当前状态 | 激活方式 |
|--------|---------|---------|
| on_tick 每tick原子链 | 解析但走空路径 | on_tick_defs 非空时执行 |
| on_window_expire 触发器 | 信号已有，未注册为触发器 | TriggerManager 监听 window_expired |
| 窗口叠加策略 | 默认 refresh | 新增 stack_mode 参数（refresh/extend/stack） |
| duration 修改 | 前置原子组合 | 同链内 modify_effect_value 修改 ctx.results |
