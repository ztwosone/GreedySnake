#### 3. Bug 修复：火蔓延导致连锁反应

**问题**: 蛇只碰冰格就被毒死。火格 `tile_effects` 有 20%/s 向邻格蔓延机制，Zone A 原布局火(row2)/冰(row4)/毒(row6) 同列，火蔓延跨行触发 steam → toxic_explosion → 毒叠 3 层 → toxify 连杀。

**修复**: 三种状态格 X 方向完全分离（火 x=3-7, 冰 x=16-20, 毒 x=30-34），火蔓延无法跨 X 到达其他类型。Zone B 反应触发区也远离火源 (>18 格)。

#### 4. Bug 修复：on_removed 链不触发 / AtomContext target 为 null / FreezeAtom 无自动恢复 / ModifySpeedAtom 未同步 timer

前一天 (03/18) 的 T25 原子系统遗留 4 个集成 bug，在本次验收测试中暴露并修复：
- `StatusEffectManager.remove_status()` 在 `_unregister_effect_chains()` 之前调用 `fire_on_removed()`
- `TriggerManager._build_context()` 同时设 `ctx.source` 和 `ctx.target` 为 carrier
- `FreezeAtom` 使用 `SceneTree.create_timer()` 实现自动恢复
- `ModifySpeedAtom` 同步 `_timer.wait_time`（已在后续重构中移除）

#### 5. 冰效果重构：全局 tick → per-entity 速度控制

**问题**: 冰减速修改 `TickManager.tick_speed_modifier`，冻结调用 `TickManager.pause()`，导致整个游戏（敌人/食物刷新等）都变慢/暂停。

**方案**: 给 Snake 加上与 Enemy 相同的 move accumulator 模式，速度来源于 `SEM.get_modifier("speed", self)`。

- **修改** `entities/snake/snake.gd` — 新增 `_move_accumulator` + `_get_effective_speed()`，从 SEM 读取 per-entity 速度修改器
- **修改** `systems/atoms/atoms/value/modify_speed_atom.gd` — 改用 `effect_mgr.set_modifier("speed", target, multiplier)`
- **修改** `systems/atoms/atoms/control/freeze_atom.gd` — `set_modifier("speed", target, 0.0)` 替代 `tick_mgr.pause()`，敌人在蛇冻结期间正常行动
- **修改** `systems/status/effects/ice_effect.gd` — 旧版也改为 per-entity modifier
- **修改** `Test/cases/test_t16_ice.gd` + `test_gameplay_sim.gd` — 测试改为验证 per-entity modifier
