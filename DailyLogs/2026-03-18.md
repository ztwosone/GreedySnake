# 2026-03-18 开发日志

## 完成任务

### T25 Effect Atom System — 效果原子系统

将硬编码的火/冰/毒 GDScript 效果逻辑升级为 JSON 驱动的可组合原子系统。

#### 框架核心（8 个文件）
- `atom_base.gd` — 原子基类（configure/execute/evaluate/is_condition）
- `atom_context.gd` — 执行上下文（source/target/positions/effect_data/results）
- `atom_registry.gd` — 49 个原子的名称→类映射，支持 `create(name, params)`
- `effect_chain.gd` — 运行时效果链（trigger + conditions + actions + pattern）
- `effect_chain_resolver.gd` — JSON → EffectChain 解析器
- `atom_executor.gd` — 链执行器（条件→概率→范围→原子）
- `pattern_resolver.gd` — 11 种空间模式解析（self/target/radius/neighbors/trail/body_segment/line/cross/ring/random_n/cone）
- `trigger_manager.gd` — 17 种触发器管理（EventBus 信号 + interval 计时器）

#### 49 个原子（9 个类别）
- **Value (9)**: damage, damage_percent, damage_cap, heal, modify_growth, modify_speed, modify_attack_cost, shield, lifesteal
- **Status (5)**: apply_status, remove_status, transfer_status, cleanse_all, extend_duration
- **Spatial (5)**: place_tile, remove_tile, place_tile_trail, convert_tile, destroy_terrain
- **Control (10)**: freeze, stun, knockback, forced_move, teleport, attract, phase, reverse_input, modify_turn_delay, lock_input
- **Spawn (3)**: spawn_entity, spawn_projectile, consume_tile
- **Temporal (4)**: delay, repeat, queue, reduce_cooldown
- **Logic/Condition (7)**: if_length_below, if_length_above, if_has_status, if_on_tile, if_chance, if_cooldown, if_count_reached
- **Build (6)**: trigger_slot, cancel_cost, disable_slot, modify_effect_value, accumulate, lock_slot

#### 系统集成
- `StatusEffectManager` — 集成 AtomRegistry + EffectChainResolver + TriggerManager；apply_status 时自动解析 JSON 原子链；新增 modifier API（get/set/clear_modifier）
- `StatusEffectData` — 新增 `chains: Array` 字段
- `LengthSystem` — 使用 SEM.get_modifier("growth") 查询增长修改器，旧 poison_effect 作为回退
- `ReactionSystem` — 支持原子链驱动的反应（优先 atoms 定义，回退旧 handler）
- `StatusTransferSystem` — trail 留痕暂由旧代码处理，未来迁移到 on_move → place_tile_trail 原子

- `_has_legacy_effects()` 逐类型判断是否需要旧处理器

## 架构说明

### 可扩展性
- 新增原子：创建 1 个 .gd 文件 + atom_registry.gd 添加 1 行
- 新增状态效果：仅修改 game_config.json，无需写 GDScript
- 新增反应：在 JSON reactions 中添加 atoms 数组即可

### 数据流
```
game_config.json → EffectChainResolver → EffectChain[]
                                              ↓
StatusEffectManager.apply_status() → TriggerManager.register_chains()
                                              ↓
EventBus signals / interval timers → TriggerManager._fire_trigger()
                                              ↓
AtomExecutor.execute_chain() → conditions → chance → PatternResolver → Atoms
```
