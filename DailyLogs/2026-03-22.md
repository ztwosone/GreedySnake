# 2026-03-22 开发日志

## 完成任务

### 游戏循环重设计 (7 阶段)

核心循环不成立（撞敌人纯亏、状态纯惩罚、敌人无法威胁蛇身），进行全面重设计。

#### Phase 1: 蛇头吃一切 + 敌人掉食物

- 蛇头碰敌人直接击杀，无长度消耗
- 敌人死前掉落可配置数量食物：wanderer=2, chaser=3, bog_crawler=4
- 所有敌人 HP 改为 1（一击必杀）
- **修改** `systems/enemy/enemy_manager.gd` — 移除 `length_decrease_requested`，新增 `_on_enemy_about_to_die`、`_spawn_food_drops`
- **修改** `systems/food_manager.gd` — 新增 `spawn_food_at(pos)`
- **修改** `scenes/game_world.gd` — 连接 `enemy_manager.food_manager`
- **修改** `data/json/game_config.json` — 所有敌人 HP=1，新增 `drop_food_count`

#### Phase 2: 逐段状态 + 全局受击计数器

- 每个 SnakeSegment 独立携带 `carried_status`（火/冰/毒/空）
- 蛇全局受击计数器：每 3 次受击掉 1 节尾巴
- 移除蛇对 StatusEffectManager 的所有依赖（速度修改器、成长修改器、视觉刷新）
- **修改** `entities/snake/snake_segment.gd` — 新增 `carried_status`、`set_carried_status()`、`clear_carried_status()`
- **修改** `entities/snake/snake.gd` — 新增 `hits_taken`、`hits_per_segment_loss`、`take_hit()`、`get_segments_in_radius()`；移除所有 StatusEffectManager 信号连接和 `_refresh_status_visuals()`；`_get_effective_speed()` 返回固定 1.0
- **修改** `systems/length/length_system.gd` — 移除 growth modifier 查询，固定 growth=1
- **修改** `autoloads/event_bus.gd` — 新增 `snake_body_attacked` 信号
- **修改** `data/json/game_config.json` — snake 新增 `hits_per_segment_loss: 3`

#### Phase 3: 敌人攻击蛇身

- 所有敌人可从相邻格攻击蛇身段（非HEAD），可配置 `can_attack`、`attack_cooldown`、`attack_range`、`attack_damage`、`prefer_attack_status`
- 攻击作为 AI 最高优先级（P0），优先于自保和追踪
- 状态传播规则：无状态→获得、同类→无事、异类→反应+双方清除
- 追踪者改为追踪最近蛇身段（而非蛇头）
- **修改** `entities/enemies/enemy_brain.gd` — 新增 `evaluate_attack()`、`_find_attackable_segment()`；`build_context()` 新增 `snake_segments` 数组
- **修改** `entities/enemies/enemy.gd` — 新增 `carried_status`、`attack_cooldown_remaining`、`_attack_segment()`、`_get_reaction_id()`；`_execute_decision()` 处理 `"attack"` 动作
- **修改** `entities/enemies/brains/chaser_brain.gd` — `evaluate_tracking()` 改为追踪最近蛇身段相邻空格；新增 `_find_nearest_body_target()`
- **修改** `data/json/game_config.json` — 所有敌人新增攻击配置字段

#### Phase 4: 状态格交互重写

- StatusTransferSystem 完全重写：蛇移动时逐段检测所在格状态格
- 交互规则：段无状态→获得、同类→无事、异类→反应+清除段状态（格保留）
- 状态格永久化（移除 duration tick）
- **重写** `systems/status/status_transfer_system.gd` — 逐段检测，移除 `_freshly_applied`、`_transfer_counters`、entity→spatial 逻辑
- **修改** `entities/status_tiles/status_tile_manager.gd` — 移除 `_process`/`tick_update`

#### Phase 5: 状态增益效果

- 火=光环：火段四邻格每 tick 伤害敌人
- 冰=防御：攻击冰段的敌人通过 StatusEffectManager 获得 ice
- 毒=尾迹：蛇尾有毒时移动留下毒格
- **新建** `systems/combat/segment_effect_system.gd` — `_process_fire_aura()`、`_process_poison_trail()`
- **修改** `entities/enemies/enemy.gd` — `_attack_segment()` 中新增冰防御
- **修改** `scenes/game_world.gd` — 连接 SegmentEffectSystem
- **修改** `data/json/game_config.json` — fire 新增 `aura_damage: 1`

#### Phase 6: 3 种异类反应

- 蒸腾 (火+冰)：范围伤敌 2 + 蛇受击 1
- 毒爆 (火+毒)：范围伤敌 3 + 蛇受击 2
- 冻疫 (冰+毒)：范围敌人获得冰+毒，蛇无伤害
- 同类状态不触发反应
- **重写** `systems/status/reaction_system.gd` — 监听 `reaction_triggered` 信号执行 AoE 效果
- **修改** `data/json/game_config.json` — 新增 `frozen_plague` 反应，所有反应新增 `enemy_damage`、`self_hit_count`

#### Phase 7: 清理 + 验收场景重设计

- 移除碾压系统全部代码（crush_system.gd 已删除）
- 验收场景 4 区布局 + 12 项自动检测清单：
  - Zone A (y=1-3): 状态拾取走廊 (火|冰|毒)
  - Zone B (y=5-8): 增益验证 (火光环|冰防御+体攻|毒尾迹)
  - Zone C (y=10-13): 反应触发 (蒸腾|毒爆|冻疫 各一条直线走廊)
  - Zone D (y=15-21): 综合战场 (6敌+散布状态格+补兵)
- **重写** `scenes/l1_acceptance.gd` — 4 区布局、12 项检测、实时 HUD (长度+受击计数)
