## 本日工作

### T27：EffectWindow 时间窗框架

完整实现 EffectWindow 时间窗系统，为触发器提供"在 N tick 内满足条件才生效"的窗口机制。

#### 新增文件
- `systems/effect/effect_window.gd` — 数据类（RefCounted），字段：`window_id`、`duration_ticks`、`rules`、`on_expire`、`cancel_on`
- `systems/effect/effect_window_manager.gd` — Node，负责窗口的开启 / 取消 / 查询，基于 tick 计时过期，过期时执行 `on_expire` atom chain，支持 `cancel_on` 信号监听

#### 新增 Atom / Condition
- `open_window` 原子：开启一个指定参数的 EffectWindow
- `if_in_window` 条件原子：检查当前是否处于指定 window_id 的活跃窗口内

#### EventBus 信号
新增 3 个信号：`window_opened`、`window_expired`、`window_cancelled`

### T28A：TriggerManager 新增 7 个触发器

扩充 TriggerManager，覆盖操作维度和资源维度的更多游戏事件。

| 触发器 | 说明 |
|--------|------|
| `on_length_change` | 蛇长变化（grow / shrink 参数区分） |
| `on_turn` | 蛇转向时触发 |
| `on_near_death` | 生命值低于阈值，每次 effect 仅触发一次（one-shot） |
| `on_streak` | 连杀计数达到指定数值时触发 |
| `on_enemy_approach` | 敌人进入指定范围时触发 |
| `on_status_gained` | 载体获得指定状态时触发 |
| `on_tile_placed` | 状态格放置到场地时触发 |

#### TriggerManager 新增状态字段
- `_turn_count`：当前 tick 内的转向次数
- `_kill_streak`：连杀计数
- `_near_death_fired`：记录已触发过 near_death 的 effect，防止重复
- `_current_tick_index`：当前全局 tick 编号

## 技术决策记录

| 决策 | 原因 |
|------|------|
| EffectWindow 使用 RefCounted 而非 Node | 纯数据容器，无需场景树生命周期，内存自动回收 |
| `on_near_death` 设计为 one-shot per effect | 防止低血量时每 tick 重复触发，需显式重置才可再次触发 |
| `_near_death_fired` 用 Dictionary 而非 Array | 以 effect_id 为 key，O(1) 查询是否已触发 |
| 移除 EffectWindow / EffectWindowManager 的 class_name | Godot 4.6 新脚本 uid 注册时序问题，与 T27A 解法一致 |
