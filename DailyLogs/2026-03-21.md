# 2026-03-21 开发日志

## 完成任务

### T26 第一层视觉反馈系统

为纯色块风格的游戏添加最低限度视觉反馈，使玩家能直观感知状态效果、碾压、受伤等核心事件。

#### 1. 蛇身状态视觉指示

蛇段携带状态时叠加对应视觉效果，多状态按优先级显示。

- 灼烧：红橙描边 + 0.5s 闪烁（`_border_rect` + Tween 循环）
- 冰冻 layer1：冰蓝色半透明覆盖层
- 冰冻 layer2（完全冻结）：白色高不透明度覆盖层
- 中毒：绿色脉动（`_status_overlay` alpha 0.15~0.45 循环）
- 优先级：冰冻 layer2 > 灼烧 > 冰冻 > 中毒
- **修改** `entities/snake/snake_segment.gd` — 新增 `_status_overlay`、`_border_rect`、`apply_status_visual()`、`clear_status_visual()`
- **修改** `entities/snake/snake.gd` — 监听状态信号，`_refresh_status_visuals()` 按优先级更新所有蛇段

#### 2. 受伤反馈

- 蛇受伤时全身红闪 0.15s（所有蛇段 `modulate` 变红再恢复）
- 长度 ≤3 时持续红色脉动，频率随长度降低加快（≤1: 0.3s, ≤2: 0.5s, ≤3: 0.8s）
- **修改** `entities/snake/snake.gd` — `_on_hurt()`、`_flash_hurt()`、`_start_danger_pulse()`、`_stop_danger_pulse()`

#### 3. 碾压视觉反馈

- 碾压命中：敌人位置白色闪光 0.3s + 屏幕微震 0.15s/2px
- 敌人死亡：Tween scale 1.0→0.0 缩小消失（替代原来的立即 queue_free）
- **新建** `systems/vfx/screen_shake.gd` — 屏幕震动管理器，监听 `snake_body_crush`
- **修改** `entities/enemies/enemy.gd` — `_play_death_animation()` 白色闪光 + 缩小消失
- **修改** `scenes/game_world.gd` — 挂载 ScreenShake

#### 4. 敌人类型形状区分

三种敌人用不同几何形状区分，形状由 JSON 配置驱动。

| 类型 | 形状 | 实现 |
|------|------|------|
| Wanderer 游荡者 | 方形 | 默认 75% ColorRect |
| Chaser 追踪者 | 菱形 | rotation=PI/4, 缩小至 70% |
| Bog Crawler 毒沼匍匐者 | 十字形 | 两个交叉 ColorRect |

- **修改** `entities/enemies/enemy.gd` — `_build_visual()` 根据 `enemy_shape` 配置切换形状

#### 5. Debug 面板

- 按 C 键切换，右上角半透明黑色面板

- 每 0.1s 自动刷新，RichTextLabel 彩色 BBCode
- **新建** `ui/debug_panel.gd`
- **修改** `scenes/game_world.gd` — 挂载 DebugPanel

#### 6. 视觉配置扩展

在 `game_config.json` 中为状态效果添加 `visual` 字段，为敌人添加 `shape` 字段。

- fire: `{ "snake_effect": "border_flash", "border_color": "#FF4C00", "flash_period": 0.5 }`
- ice: `{ "snake_effect": "overlay", "overlay_color": "#80B0FF66", "freeze_overlay_color": "#E6F0FFB3" }`
- poison: `{ "snake_effect": "pulse", "overlay_color": "#33CC1A4D", "pulse_period": 1.0 }`
- 敌人 shape: wanderer="square", chaser="diamond", bog_crawler="cross"
