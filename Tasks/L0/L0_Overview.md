# L0-MVP 任务总览

**目标：** 一条蛇在一个房间里能移动、吃食物、撞敌人、死亡、重开。完整的可运行 Gameplay Loop。

---

## 任务清单

| 编号 | 任务名 | 优先级 | 粒度 | 前置 | 职能 |
|------|--------|--------|------|------|------|
| T01 | 项目脚手架与核心常量 | P0 阻塞 | S | 无 | 引擎程序 |
| T02 | EventBus 全局事件总线 | P0 阻塞 | S | T01 | 引擎程序 |
| T03 | TickManager 节拍管理器 | P0 阻塞 | M | T01, T02 | 引擎程序 |
| T04 | GridWorld 网格世界管理器 | P0 阻塞 | L | T01, T02 | 引擎程序 |
| T05 | GridEntity 万物基类 | P0 阻塞 | M | T01, T04 | 引擎程序 |
| T06 | Snake 实体与移动系统 | P1 核心 | XL | T02~T05 | Gameplay |
| T07 | LengthSystem 长度系统 | P1 核心 | M | T02, T06 | Gameplay |
| T08 | Food 食物系统 | P1 核心 | M | T04, T05 | Gameplay |
| T09 | 静止敌人与基础战斗 | P2 重要 | M | T02, T04, T05, T07 | Gameplay |
| T10 | 游戏场景组装与视觉层 | P1 核心 | L | T01~T09 | Gameplay |
| T11 | GameManager 与游戏循环 | P1 核心 | L | T02, T10 | Gameplay |

## 依赖关系图

```
T01 (脚手架)
 ├── T02 (EventBus)
 │    ├── T03 (TickManager)
 │    └──┐
 ├── T04 (GridWorld)
 │    └── T05 (GridEntity)
 │         ├── T06 (Snake + Movement) ← 也依赖 T02, T03
 │         │    └── T07 (LengthSystem)
 │         ├── T08 (Food)
 │         └── T09 (Enemy + Combat) ← 也依赖 T02, T07
 │
 └── T10 (场景组装) ← 依赖 T01~T09 全部
      └── T11 (GameManager + 游戏循环)
```

## 可并行执行的任务组

| 阶段 | 可并行任务 | 说明 |
|------|-----------|------|
| 阶段 1 | T01 | 前置条件，必须最先完成 |
| 阶段 2 | T02, T04 | 互相无依赖，可并行 |
| 阶段 3 | T03, T05 | T03 依赖 T02；T05 依赖 T04 |
| 阶段 4 | T06, T08 | T06 依赖 T02~T05；T08 依赖 T04+T05；可并行 |
| 阶段 5 | T07, T09 | T07 依赖 T06；T09 依赖 T05+T07；T09 需等 T07 |
| 阶段 6 | T10 | 集成任务，等待全部前置 |
| 阶段 7 | T11 | 最终任务，游戏循环 |

## MVP 验收标准（全部任务完成后）

- [ ] 启动游戏显示标题画面
- [ ] 点击开始后，蛇在 Grid 上按 0.25s/tick 移动
- [ ] WASD / 方向键控制转向
- [ ] 地图上有 3 个食物和 3 个敌人
- [ ] 吃食物 → 蛇变长，HUD 长度数字更新
- [ ] 撞敌人 → 敌人消失，蛇变短，HUD 更新
- [ ] 撞墙/撞自身 → Game Over 画面
- [ ] 长度归零 → Game Over 画面
- [ ] Game Over 显示分数和死因
- [ ] 点击重开 → 全新一局
- [ ] 反复重开无异常
