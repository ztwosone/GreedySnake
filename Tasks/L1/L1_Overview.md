# L1-Combat 任务总览

**目标：** 战斗变得有趣。状态效果系统上线，敌人有 AI 行为差异，敌人主动攻击蛇身，战场上出现元素反应。

**前置：** L0-MVP 全部完成（T01–T11）

**状态：** ✅ **全部完成**（T12a–T26 共 16 项任务，1017 测试通过，2026-03-24）

> **L1 里程碑总结：** 原设计的「蛇身碾压」在原型验证中调整为「敌人主动攻击蛇身 + 双向状态转移」模型。核心玩法循环：蛇吃敌人（无长度消耗）→ 掉食物 → 蛇身段独立携带状态 → 敌人攻击蛇身（3 次受击掉 1 段）→ 双向状态传递/反应触发。验收场景含 12 项自动检测。

## 各任务简述

### T12a — JSON 可配置参数系统 ✅
- `data/json/game_config.json`：L0 参数 + L1 状态效果/敌人/反应的完整配置
- `autoloads/config_manager.gd`：加载 JSON，提供 `get_status_effect()` / `get_enemy_type()` / `find_reaction()` 等类型化 API
- `core/constants.gd` 保守迁移：保留 `const` 编译期常量，新增 `var` 运行时别名从 ConfigManager 读取
- `Tools/config_editor.html`：独立 HTML 配置编辑器，可视化 / JSON 源码双模式
- 测试：54 项通过（总计 338/338）

### T12 — StatusEffect 数据模型与管理器
- `StatusEffect` 数据类：type, layer, carrier, duration, source
- `StatusEffectManager` 单例/系统节点：管理所有活跃的状态实例
- 状态的添加、叠层、过期移除
- EventBus 信号：`status_applied`, `status_removed`, `status_layer_changed`

### T13 — StatusTile 空间状态格
- `StatusTile` 继承 `GridEntity`：代表地板上的状态格
- 有 type, layer, duration, 视觉颜色
- 到期自动消失
- 实体踩入时触发 `entity_entered_status_tile` 信号

### T14 — 实体↔空间状态转化系统
- 蛇身带状态经过空格 → 在该格生成对应 StatusTile（实体→空间）
- 实体移动到 StatusTile 上 → 实体获得对应状态（空间→实体）
- 转化规则由各状态类型定义，通过配置表驱动

### T15 — 火焰状态实现
- 实体效果：每 2 秒 -1 格长度，持续 6 秒，可叠层
- 空间效果：火焰格持续 8 秒，经过的实体获得灼烧
- 视觉：橙红色 ColorRect + 简单闪烁

### T16 — 冰冻状态实现
- 实体效果：移速 -50%，叠加第 2 层时完全静止 2 秒
- 空间效果：冰霜格持续 12 秒，经过时获得冰冻
- 视觉：蓝白色 ColorRect

### T17 — 中毒状态实现
- 实体效果：食物增长量减半，持续 8 秒，3 层触发毒化
- 空间效果：毒液格持续 10 秒，阻止食物生成
- 视觉：深绿色 ColorRect

### T18 — 状态反应系统 + 蒸腾/毒爆
- 反应检测：同一实体/相邻格存在两种不同状态时触发
- 反应框架：反应表（type_a × type_b → reaction_def）
- 蒸腾（火+冰）：周围 3 格生成蒸汽云，系数 0.5
- 毒爆（火+毒）：范围 3 格附加灼烧×2+中毒×1，系数 1.0

### T19 — 敌人 AI 行为框架
- `EnemyBrain` 基类：行为优先级栈（P1 自保 → P2 威胁 → P3 状态响应 → P4 追踪 → P5 默认）
- 状态响应类型枚举：回避/无视/趋向/利用/恐惧
- 寻路辅助：简单 Manhattan 距离 / 可选 A* 网格寻路
- 每 tick 由 TickManager 驱动执行

### T20 — 游荡者（Wanderer）
- 状态响应：无视型
- 默认行为：随机移动，碰壁反弹
- HP：1 格
- 替换 L0 的静止敌人

### T21 — 追踪者（Chaser）
- 状态响应：回避型（绕开状态格）
- 追踪：每 tick 向蛇头移动 1 格
- 威胁响应：蛇头进入 3 格范围内，移速 +1
- HP：1~2 格

### T22 — 毒沼匍匐者（Bog Crawler）
- 状态响应：趋向型（偏好毒液格）
- 特殊：在毒液格上移速 +2；死亡时爆裂留下 3 格毒液
- HP：2 格

### T23 — ~~蛇身碾压系统~~ → 敌人攻击蛇身（DEPRECATED）
- ~~蛇移动时，身体段经过的格子如果有敌人 → 触发碾压判定~~
- **实际实现：** 碾压机制移除，改为敌人主动攻击蛇身段（3 次受击丢 1 段）+ 双向状态转移
- 详见 `enemy.gd:_attack_segment` + `status_transfer_system.gd`

### T24 — 战斗场景集成与平衡调试
- EnemyManager 支持多种敌人类型混合生成
- 房间内敌人类型 ≤ 3 种（设计文档要求）
- 状态格视觉层（渲染在 grid 上方、实体下方）
- 初步数值平衡：食物/敌人/状态效果的数量与强度
- HUD 扩展：显示当前蛇身状态

### T25 — Effect Atom System（效果原子系统）✅
- JSON 驱动的可组合效果框架，替代硬编码 GDScript 效果逻辑
- 49 个原子（9 类别：Value/Status/Spatial/Control/Spawn/Temporal/Logic/Build）
- 17 种触发器 + 11 种空间模式
- 旧 FireEffect/IceEffect/PoisonEffect 保留为 shim 回退
- 248 个新增测试，总计 1053 测试通过

### T26 — 第一层视觉反馈系统
- 蛇身状态视觉指示（灼烧描边闪烁/冰冻变蓝/中毒绿色脉动）
- ~~碾压反馈~~（碾压已移除，改为敌人攻击蛇身反馈）
- 敌人类型形状区分（方形/菱形/十字形）
- 受伤反馈（全身红闪 + 低血量脉动）
- 设计文档：`Designs/Interactive/visual_feedback_design.md`
- L2 开始前的最后一个必要任务
